## Contenido del módulo — Semana 7: Datos y análisis

### 1. Anatomía de un dataset

Un **dataset** (conjunto de datos) tiene estructura tabular: filas y columnas.

- **Filas = observaciones**: cada fila es un registro individual (una persona, un evento, una ciudad, una transacción).
- **Columnas = variables**: cada columna es un atributo o característica de esa observación.
- **Celda = valor**: la intersección de una fila y una columna contiene exactamente un valor.

**Tipos de variables frecuentes:**

| Tipo | Subtipo | Ejemplo |
|---|---|---|
| Numérica | Continua | Temperatura (20.5 °C), precio ($349.90) |
| Numérica | Discreta | Número de hijos (0, 1, 2…), cantidad de clicks |
| Categórica | Nominal | País, color, género, nombre de carrera |
| Categórica | Ordinal | Nivel de satisfacción (1–5), grado escolar |
| Texto libre | — | Comentario de reseña, descripción de producto |
| Fecha/hora | — | 2024-03-15, 14:30:00 |

**¿Qué es un dataset "tidy"?**

Un dataset está ordenado (_tidy_) cuando cumple tres reglas:
1. Cada fila es exactamente una observación.
2. Cada columna es exactamente una variable.
3. Cada celda contiene exactamente un valor.

Los datasets reales casi nunca llegan así. Limpiarlos es parte del trabajo.

**Formatos comunes:**

- **CSV** (Comma-Separated Values): texto plano, universal, fácil de compartir. La primera fila suele ser el encabezado.
- **Excel / ODS**: formato de hoja de cálculo con múltiples pestañas. Cuidado con celdas combinadas y macros ocultas.
- **JSON**: estructurado como pares clave-valor, frecuente en APIs. Puede ser anidado (difícil de convertir a tabla).

### 2. El ciclo de análisis de datos

El análisis de datos no es un paso único: es un ciclo de seis etapas. Los asistentes de IA pueden ayudar en cada una, pero el juicio en cada etapa es humano.

**Etapa 1 — Definir la pregunta**  
¿Qué quieres saber exactamente? Una buena pregunta analítica es específica, responde a un interés real y puede contestarse con los datos disponibles. Ejemplo malo: "analizar las ventas". Ejemplo bueno: "¿Cuál es el promedio de ventas mensuales por región entre enero y junio de 2023?"

**Etapa 2 — Obtener los datos**  
¿De dónde provienen? ¿Qué licencia tienen? ¿Son relevantes para la pregunta? Documenta la URL o DOI, la fecha de descarga y la versión. Sin esto, el análisis no es reproducible.

**Etapa 3 — Limpiar los datos**  
Eliminar duplicados, corregir tipos (texto donde debería haber número), estandarizar formatos de fecha, manejar valores faltantes (¿eliminar la fila? ¿rellenar con la mediana? ¿marcarlo como ausente?). Cada decisión de limpieza debe documentarse.

**Etapa 4 — Analizar**  
Calcular estadísticas descriptivas (media, mediana, desviación estándar, frecuencias). Crear visualizaciones para detectar patrones. Cruzar variables. En esta etapa la IA puede generar fórmulas o código; tú decides qué calcular.

**Etapa 5 — Interpretar**  
¿Qué significan los números? ¿Qué no dicen? Correlación no implica causalidad. Un promedio puede ocultar variaciones importantes. Un resultado estadístico puede ser real pero no importante en la práctica. Los límites de interpretación deben declararse explícitamente.

**Etapa 6 — Comunicar**  
¿Quién es la audiencia? ¿Qué nivel de detalle necesita? ¿Qué formato es más claro (tabla, gráfica de barras, mapa)? Una visualización sin etiquetas o sin fuente no es comunicación, es ruido.

### 3. Limpieza de datos con IA (asistida)

Puedes describir problemas de calidad de datos a un asistente de IA y pedir fórmulas o código para resolverlos. Lo importante es entender el principio:

**Lo que puedes pedirle a la IA:**

- "Tengo una columna de fechas con formato DD/MM/AAAA y AAAA-MM-DD mezclados. ¿Cómo estandarizo en Google Sheets?"
- "¿Qué fórmula de Excel elimina los espacios dobles en una columna de nombres?"
- "¿Cómo cuento en Python cuántas filas tienen valor nulo en la columna 'edad'?"

**Lo que tú debes hacer después:**

1. Leer la fórmula o el código que te dé antes de aplicarlo.
2. Verificar manualmente con 3 a 5 filas que conoces. ¿El resultado es el esperado?
3. Documentar: qué pediste, qué fórmula obtuviste, qué modificaste, qué validaste.

**Lo que la IA no puede hacer:**

- Ver tu archivo real (a menos que lo pegues; y si tiene datos personales, no debes hacerlo).
- Saber qué significa un valor faltante en tu contexto: ¿es un cero? ¿Un dato no recabado? ¿Un error?
- Decidir si eliminar filas con valores ausentes o imputarlos. Esa decisión metodológica es tuya.

**Regla de privacidad:**  
Para datasets de práctica (≤ 100 filas, sin datos personales identificables): puedes pegar el CSV directamente en el chat de IA. Para datasets más grandes o con cualquier información personal (nombres, correos, ubicaciones, identificadores): usa la IA para generar el código y aplícalo localmente. Nunca subas datos de personas reales a servicios externos sin revisión de privacidad institucional.

### 5. Documentación de datos: el README de datos

Todo análisis académico o profesional que use un dataset externo necesita un README de datos. Sin él, el análisis no es reproducible ni defendible.

**Campos mínimos requeridos:**

```
# README de datos — [Nombre del análisis]

## Fuente
- Nombre del dataset: ...
- URL o DOI: ...
- Organización/autor: ...

## Licencia
- Tipo: [CC0 / CC-BY 4.0 / CC-BY-SA / Propietaria / Otra]
- ¿Puedo usarlo para este trabajo?: [Sí / No / Con restricciones]
- Cómo citar: ...

## Acceso
- Fecha de descarga: AAAA-MM-DD
- Versión o timestamp del archivo: ...

## Descripción del dataset original
- Filas: N
- Columnas: M
- Período cubierto: ...
- Unidad de observación: ...

## Modificaciones realizadas
- Columnas eliminadas: [listar]
- Filas eliminadas: [criterio]
- Valores imputados o corregidos: [qué, cómo, por qué]
- Formatos normalizados: [ej. fechas estandarizadas a ISO 8601]

## Limitaciones conocidas
- [Sesgo de muestreo, grupos subrepresentados, período desactualizado, etc.]

## Uso de IA en la limpieza
- [Qué fórmulas o código generó la IA, qué se verificó manualmente]
```

**Por qué importa:**  
Sin documentación, nadie puede saber si tu análisis es correcto. Si tus datos cambian, no sabrás si tus resultados cambian con ellos. Si alguien encuentra un error, no podrá rastrear su origen. La documentación no es burocracia: es parte del análisis.

### 6. Asistentes de código: qué hacen (y qué no hacen)

Los asistentes de código basados en IA (GitHub Copilot, ChatGPT, Claude, Gemini) pueden hacer varias cosas útiles:

| Capacidad | Descripción | Ejemplo |
|---|---|---|
| **Autocompletar** | Sugiere la siguiente línea basándose en el contexto | Escribes `df.groupby(` y sugiere el resto |
| **Generar desde descripción** | Escribe una función a partir de un comentario | "# Lee un CSV y devuelve filas donde edad > 18" |
| **Explicar código** | Describe en lenguaje natural qué hace un bloque | Pegás código y preguntás "¿qué hace esto?" |
| **Revisar y mejorar** | Señala bugs potenciales o mejoras de estilo | "¿Hay algún problema en esta función?" |
| **Traducir entre lenguajes** | Convierte Python a R, SQL a Python, etc. | "Conviérteme esta consulta SQL a pandas" |

**Límites críticos:**

- La IA no conoce tu entorno específico (versión de Python, librerías instaladas, estructura de tu proyecto).
- La IA no conoce tu dataset real ni sus particularidades.
- La IA puede generar código que parece correcto pero falla con tus datos reales.
- La IA puede usar funciones obsoletas o con comportamiento diferente entre versiones.
- La IA no puede ejecutar el código: eso siempre lo haces tú.

### 7. Anatomía de un Jupyter Notebook

Un notebook de Jupyter (o Google Colab) alterna dos tipos de celdas:

**Celdas de código:**  
Contienen Python (o R) ejecutable. El resultado aparece debajo al correr la celda. Cada celda tiene un número de ejecución `[1]`, `[2]`... que indica el orden en que se corrió.

**Celdas de Markdown:**  
Contienen texto formateado (títulos, listas, explicaciones, notas de interpretación). No se ejecutan; se renderizan como texto.

**Buena práctica para un notebook reproducible:**

1. Toda celda de código computacionalmente importante tiene una celda de Markdown encima que explica **por qué** se hace ese cálculo (no solo qué hace el código).
2. El notebook se puede correr de arriba a abajo sin errores (menú: Kernel → Restart & Run All).
3. Las variables intermedias tienen nombres descriptivos, no `x`, `tmp`, `data2`.
4. Los resultados numéricos tienen unidades en la celda de interpretación adyacente.

**Ejemplo de estructura correcta:**

```
[Markdown] ## 2. Limpieza de datos
[Markdown] Eliminamos filas con valor nulo en la columna 'ingreso'
            porque representan registros incompletos (ver README).
            Total de filas antes: 200. Después: 187.
[Código]   df_clean = df.dropna(subset=['ingreso'])
[Código]   print(f"Filas eliminadas: {len(df) - len(df_clean)}")
[Markdown] **Interpretación:** Se eliminaron 13 filas (6.5% del total).
            Este porcentaje es bajo; no esperamos sesgo significativo.
```

### 8. Flujo de trabajo con asistente de código

El flujo en seis pasos garantiza que el código generado por IA sea tuyo en términos de comprensión y responsabilidad:

**Paso 1 — Escribir el comentario**  
Antes de pedir a la IA, escribe un comentario que describe exactamente qué quieres que haga la función. Si no puedes describirlo con claridad, no estás listo para pedirlo.

```python
# Calcula el promedio de la columna 'precio' agrupado por 'categoria'
# Devuelve un DataFrame con columnas: categoria, precio_promedio
# Los valores nulos en 'precio' deben ignorarse (skipna=True)
```

**Paso 2 — Pedir a la IA (o dejar que autocomplete)**  
Usa tu comentario como prompt. Pide a la IA que genere el código o deja que el asistente autocomplete.

**Paso 3 — Leer el código línea por línea antes de ejecutar**  
No ejecutes código que no hayas leído. Para cada línea pregúntate: ¿Sé qué hace esto? ¿Tiene sentido aquí? Si hay algo que no entiendes, pide explicación antes de correr.

**Paso 4 — Probar con entradas conocidas**  
Crea un mini-dataset de prueba donde conoces la respuesta esperada. Verifica que el código produce ese resultado.

```python
# Dataset de prueba
test_df = pd.DataFrame({
    'categoria': ['A', 'A', 'B', 'B'],
    'precio': [10, 20, 30, None]
})
# Resultado esperado: A=15.0, B=30.0
resultado = calcular_promedio_por_categoria(test_df)
print(resultado)
```

**Paso 5 — Manejar casos borde**  
¿Qué pasa si la columna está vacía? ¿Si el DataFrame tiene cero filas? ¿Si el tipo de dato es texto en vez de número? Prueba al menos dos casos extremos.

**Paso 6 — Documentar**  
En la celda de Markdown del notebook, registra: qué prompt usaste, qué código generó la IA, qué modificaste y qué pruebas corriste. Esto es trazabilidad.

### 9. Cómo leer código generado por IA

La regla de oro: **si no puedes explicar cada línea, no puedes defenderla.**

**Proceso de lectura:**

1. Lee la firma de la función: ¿qué recibe? ¿qué devuelve?
2. Lee el cuerpo línea por línea.
3. Traza una entrada de ejemplo a mano: ¿qué valor tiene cada variable en cada paso?
4. Pregunta a la IA que explique las líneas que no entiendes (sin copiar la explicación sin leerla).

**Señales de alerta en código generado — revisar con cuidado:**

| Señal | Por qué preocupa |
|---|---|
| `eval()` o `exec()` | Ejecuta código arbitrario; peligro de inyección |
| `subprocess` o `os.system()` | Ejecuta comandos del sistema operativo |
| `requests.get()` sin explicación | Hace llamadas a internet desde tu entorno |
| `open()` con modo `'w'` en ruta absoluta | Escribe o sobreescribe archivos en tu sistema |
| Importaciones de librerías desconocidas | Pueden requerir datos o permisos adicionales |

Ante cualquiera de estas señales: pregunta a la IA que explique exactamente qué hace esa línea y por qué la incluyó. Si la explicación no te convence, elimina o reemplaza esa parte.

### 10. Visualización de datos responsable

Una visualización sin los elementos correctos es desinformación potencial, aunque sea involuntaria.

**Elementos obligatorios en toda visualización:**

1. **Título** claro: qué muestra la gráfica (no "Gráfica 1", sino "Distribución de calificaciones por semestre, 2023")
2. **Etiquetas de ejes**: nombre de la variable + unidad si aplica
3. **Fuente**: de dónde vienen los datos (con URL o referencia completa)
4. **N**: tamaño de la muestra (n = 187 observaciones)
5. **Nota de límites**: una oración que declare qué NO muestra la gráfica o qué sesgo puede tener

**Errores comunes a evitar:**

- **Eje Y truncado:** empezar el eje Y en un valor distinto de cero exagera visualmente las diferencias.
- **Escala engañosa:** usar escala logarítmica sin avisarlo.
- **Rango de fechas cherry-picked:** mostrar solo el período que favorece tu argumento.
- **Gráficas de pay para comparaciones complejas:** difíciles de leer con más de 5 categorías.
- **Colores sin significado:** usar rojo/verde para diferencias que no son negativas/positivas.

**La IA puede generar el código de la gráfica. Tú debes:**
- Verificar que las etiquetas son correctas y en el idioma correcto.
- Confirmar que la escala es apropiada y no engañosa.
- Asegurarte de que la visualización representa fielmente los datos.
- Añadir la nota de límites tú mismo (la IA no sabe qué sesgo tiene tu dataset).

## Ruta sin código (hoja de cálculo)

Esta ruta cubre exactamente los mismos objetivos de aprendizaje que la ruta con Python. No es una versión reducida: es una alternativa completa para quien no programa o prefiere una herramienta visual.

### Herramientas válidas

- **Google Sheets** (en línea, gratuito con cuenta Google)
- **Microsoft Excel** (escritorio o Microsoft 365)
- **LibreOffice Calc** (gratuito, funciona sin internet)

### Lo que harás en hoja de cálculo

| Equivalente en Python | Equivalente en hoja de cálculo |
|---|---|
| `df.head()` | Ver las primeras filas visualmente |
| `df.describe()` | Funciones `PROMEDIO()`, `MAX()`, `MIN()`, `CONTAR()` |
| `df.groupby('col').mean()` | Tabla dinámica (pivot table) |
| `df.dropna()` | Filtrar y eliminar filas vacías manualmente |
| `matplotlib.pyplot.bar()` | Insertar → Gráfico → Barras |
| `df['col'].value_counts()` | `CONTAR.SI()` por categoría |

### Cómo usar IA en la ruta de hoja de cálculo

1. **Describir el problema en lenguaje natural:** "Tengo una columna A con categorías y una columna B con números. Quiero el promedio de B para cada valor único de A en Google Sheets."
2. **Recibir la fórmula:** La IA podría responder: `=PROMEDIO.SI(A:A,"Categoría1",B:B)`
3. **Verificar manualmente:** Calcula a mano el promedio de 3 o 4 filas de "Categoría1" y compara con el resultado de la fórmula.
4. **Documentar:** En una celda de comentario o en un documento aparte, registra qué pediste, qué fórmula obtuviste, qué validaste.

## Guía de sesión síncrona — Semana 7

**Duración total:** 90 minutos  
**Tema:** Datos, hojas de cálculo y análisis asistido

### Apertura (10 min)

Pregunta disparadora: "¿Alguna vez descargaste datos y no sabías si podías usarlos? ¿Qué hiciste?"

Conectar con el módulo anterior (escritura y autoría): así como en escritura declaramos qué generó la IA y qué escribimos nosotros, en análisis declaramos qué calculó la IA y qué decidimos nosotros.

Anunciar las dos rutas disponibles y confirmar cuántos estudiantes seguirán cada una.

### Demo guiada (15 min)

**Instructor:** Abrir el CSV provisto en Google Sheets en pantalla compartida.  
Mostrar:
- Cómo identificar los tipos de variables de cada columna.
- Cómo hacer una pregunta analítica simple ("¿Cuál es el promedio de la columna X por categoría?").
- Cómo pedirle a un asistente de IA la fórmula o el código.
- Cómo verificar el resultado manualmente con 3 filas.

**Punto de énfasis:** El instructor muestra explícitamente el paso de verificación manual. Sin esto, el ejercicio no enseña nada.

### Práctica estructurada (20 min)

Estudiantes cargan el CSV en su herramienta. El instructor circula (en clase presencial) o responde en el chat (en línea).

**Consigna:** "Identifica el tipo de cada variable en el dataset. Responde: ¿cuántas observaciones hay? ¿Cuántas variables numéricas? ¿Hay valores faltantes visibles?"

Pausa de 2 min para que compartan respuestas en chat o en voz alta.

### Transición (5 min)

Introducir el concepto de limpieza: "Ahora que conocemos el dataset, ¿qué problemas de calidad vemos?" Anotar respuestas del grupo en pantalla.

### Práctica aplicada (25 min)

## Preguntas frecuentes del estudiantado

**"No sé Python. ¿Puedo aprobar este módulo?"**  
Sí. La ruta sin código cubre exactamente los mismos objetivos de aprendizaje. Lo que se evalúa es la comprensión del flujo de análisis, la documentación y el pensamiento crítico sobre los resultados, no el lenguaje de programación usado.

**"El código que me dio la IA no funciona. ¿Qué hago?"**  
Eso es esperado y es parte del aprendizaje. Proceso: (1) Lee el mensaje de error completo; (2) Identifica la línea que lo produce; (3) Busca ese mensaje de error en Google o pregúntale a la IA específicamente sobre ese error; (4) Prueba paso a paso, no ejecutes todo de golpe. El debugging es una habilidad, no un fracaso.

**"Encontré un dataset en Kaggle. ¿Puedo usarlo?"**  
Sí, si verificas la licencia. Ve a la pestaña del dataset → sección "License". CC0 = libre uso sin restricciones. CC-BY = libre uso con atribución obligatoria. "Other" o sin licencia especificada = tratar como restringido y consultar. Documenta lo que encuentres en el README.

**"¿Puedo pegar todo mi CSV en ChatGPT para que lo analice?"**  
Depende del dataset. Si tiene ≤ 100 filas y cero datos personales identificables: sí, con declaración de uso de IA. Si tiene más filas o cualquier dato personal (nombres, correos, teléfonos, identificadores): no. Usa la IA para generar el código y aplícalo localmente. Esta distinción no es opcional.

**"¿La IA puede equivocarse en estadísticas?"**  
Sí. Puede calcular mal, usar la función incorrecta o malinterpretar tu pregunta. Por eso el paso de verificación manual existe. Nunca reportes un resultado que no hayas comprobado por otra vía.

**"¿Cómo sé si mi visualización está bien?"**  
Checklist: ¿Tiene título descriptivo? ¿Los ejes tienen etiquetas y unidades? ¿Se indica la fuente de los datos? ¿Se indica el n (tamaño de muestra)? ¿Hay una nota que declare qué no muestra la gráfica? Si las cinco respuestas son sí, la visualización cumple los requisitos mínimos.

**"¿Puedo usar un dataset sobre datos de personas reales que encontré en internet?"**  
Solo si tiene una licencia que lo permite explícitamente y siempre y cuando no publiques ni redistribuyas los datos individuales. Datos agregados (por región, por año, por grupo) generalmente son seguros. Datos a nivel de individuo requieren revisión cuidadosa. Ante la duda, usa uno de los datasets provistos o aprobados por el instructor.

## Señales de alerta y cómo responder

| Señal observada | Qué está pasando | Cómo responder |
|---|---|---|
| El estudiante copia código sin leerlo ni ejecutarlo | Está usando la IA como sustituto, no como asistente | Pedir que explique cada línea en voz alta. Si no puede, debe leer antes de avanzar. |
| El análisis no tiene README ni fuente de datos | No hay trazabilidad | Pedir que complete el README antes de aceptar cualquier otra entrega. Sin documentación no hay entrega válida. |
| La visualización no tiene título ni ejes | Resultado incompleto | Mostrar la lista de elementos obligatorios. Pedir corrección antes de la entrega final. |
| El estudiante reporta resultados sin haber verificado manualmente | Riesgo de error sin detectar | Pedir que muestre el cálculo manual de verificación para al menos 3 filas. |
| El estudiante pega datos personales en la IA | Violación de privacidad | Detener inmediatamente. Explicar la regla. Pedir que documente el incidente si hay datos sensibles involucrados. Escalar a coordinación si corresponde. |
| El estudiante dice "la IA no puede equivocarse en matemáticas" | Malentendido sobre las capacidades de la IA | Mostrar un ejemplo concreto de error estadístico de la IA. La IA puede usar la función equivocada, malinterpretar el contexto o devolver resultados plausibles pero incorrectos. |
| El notebook no corre de arriba a abajo sin errores | No es reproducible | Pedir que ejecute Kernel → Restart & Run All y corrija todos los errores antes de entregar. |
| El código generado por IA incluye `eval()`, `subprocess` u otras llamadas de riesgo | Posible riesgo de seguridad | Pedir al estudiante que pregunte a la IA qué hace esa línea y si puede reemplazarse por algo más simple. No ejecutar ese código hasta que se entienda. |
